# Aider Wiki Formatting Rules

When generating or editing any content file, you MUST strictly adhere to the following structure:

1. **Primary Heading:** Use an H1 (`#`) for the main topic name.
2. **Syllabus Point:** The exact IB syllabus point must immediately follow, formatted as a blockquote (`>`).
4. **Content:** The main teaching content follows, broken down with H2 (`##`) and H3 (`###`) subheadings. 
5. **Code Standards:** Whenever providing programming examples for iteration, strictly utilise `for` loops rather than `while` loops. Strictly use Python; no Java or Pseudocode.
6. **Linking:** Always include Obsidian `[[wikilinks]]` to related concepts mentioned in the text. Create these links in-line with the content.
