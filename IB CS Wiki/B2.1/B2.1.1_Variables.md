# Variables and Data Types

> B2.1.1 Construct and trace programs using a range of global and local variables of various data types. Data types: Boolean, char, decimal, integer, string.

## What Are Variables?

Variables are named containers that store data in your program. Think of them as labelled boxes where you can put different types of information. The [[Central_Processing_Unit|CPU]] uses registers to temporarily hold these values during program execution.

When you create a variable, you're telling the computer to reserve a specific location in [[Primary_Memory|RAM]] to store that piece of data. Each variable has:

- **A name** (identifier) - what you call it in your code
- **A data type** - what kind of data it holds
- **A value** - the actual data stored
- **A scope** - where in the program it can be accessed

## Data Types in Python

Python supports several fundamental data types that you need to understand:

### Integer (`int`)

Integers are whole numbers without decimal points. They can be positive, negative, or zero.

```python
# Integer examples
age = 16
temperature = -5
score = 0
year = 2024

# Using integers in calculations
total_students = 25
absent_students = 3
present_students = total_students - absent_students
print(f"Students present: {present_students}")  # Output: Students present: 22
```

### Decimal (`float`)

Floating-point numbers (floats) represent numbers with decimal points. They're essential for precise calculations.

```python
# Float examples
pi = 3.14159
temperature = 36.6
price = 19.99
percentage = 87.5

# Using floats in calculations
distance = 100.5  # kilometres
time = 2.25  # hours
speed = distance / time
print(f"Average speed: {speed:.2f} km/h")  # Output: Average speed: 44.67 km/h
```

### String (`str`)

Strings store text data - sequences of characters enclosed in quotes (single or double).

```python
# String examples
name = "Alice"
greeting = 'Hello, World!'
subject = "Computer Science"
empty_string = ""

# String operations
first_name = "John"
last_name = "Smith"
full_name = first_name + " " + last_name  # Concatenation
print(full_name)  # Output: John Smith

# String length
message = "IB Computer Science"
length = len(message)
print(f"Message length: {length}")  # Output: Message length: 19
```

### Boolean (`bool`)

Booleans represent truth values: either `True` or `False`. They're crucial for decision-making in programs.

```python
# Boolean examples
is_student = True
has_passed = False
is_raining = True

# Boolean expressions
age = 17
is_adult = age >= 18
print(f"Is adult: {is_adult}")  # Output: Is adult: False

# Using booleans in conditions
temperature = 25
is_warm = temperature > 20
if is_warm:
    print("It's a warm day!")
```

### Character (`char`)

Python doesn't have a separate character type - a single character is simply a string of length 1.

```python
# Character examples (stored as strings in Python)
grade = 'A'
initial = 'J'
symbol = '@'

# Checking if a string is a single character
letter = 'X'
if len(letter) == 1:
    print(f"{letter} is a single character")
```

## Variable Scope: Global vs Local

Understanding scope is essential for writing organised, bug-free code. Scope determines where a variable can be accessed in your program.

### Local Variables

Local variables are defined inside a function and can only be used within that function. They're created when the function is called and destroyed when it finishes.

```python
def calculate_area():
    # 'length' and 'width' are local variables
    length = 10
    width = 5
    area = length * width
    return area

result = calculate_area()
print(f"Area: {result}")  # Output: Area: 50

# This would cause an error - 'length' doesn't exist outside the function
# print(length)  # NameError: name 'length' is not defined
```

### Global Variables

Global variables are defined outside all functions and can be accessed from anywhere in the program.

```python
# 'school_name' is a global variable
school_name = "International School"
student_count = 500

def display_school_info():
    # We can read global variables inside functions
    print(f"School: {school_name}")
    print(f"Students: {student_count}")

display_school_info()
# Output:
# School: International School
# Students: 500

# Global variables can be accessed outside functions too
print(f"Welcome to {school_name}!")
```

### Modifying Global Variables

To modify a global variable inside a function, you must use the `global` keyword.

```python
# Global variable
total_score = 0

def add_points(points):
    global total_score  # Declare we're using the global variable
    total_score = total_score + points
    print(f"Added {points} points. Total: {total_score}")

add_points(10)  # Output: Added 10 points. Total: 10
add_points(25)  # Output: Added 25 points. Total: 25
add_points(15)  # Output: Added 15 points. Total: 40

print(f"Final score: {total_score}")  # Output: Final score: 40
```

### Best Practices for Scope

```python
# GOOD: Use local variables when possible
def calculate_discount(price, discount_rate):
    # Local variables keep the function self-contained
    discount_amount = price * discount_rate
    final_price = price - discount_amount
    return final_price

# AVOID: Overusing global variables
# Global variables can make code harder to debug and maintain
counter = 0  # Global

def increment():
    global counter
    counter += 1  # Modifying global state

# BETTER: Pass values as parameters and return results
def increment_value(value):
    return value + 1

counter = 0
counter = increment_value(counter)  # Clearer data flow
```

## Tracing Programs with Variables

Tracing means following the execution of a program step-by-step to understand how variable values change. This is a crucial debugging skill.

### Example 1: Tracing with Local Variables

```python
def process_scores():
    score1 = 85  # Step 1: score1 = 85
    score2 = 92  # Step 2: score2 = 92
    score3 = 78  # Step 3: score3 = 78
    
    total = score1 + score2 + score3  # Step 4: total = 255
    average = total / 3  # Step 5: average = 85.0
    
    return average

result = process_scores()  # Step 6: result = 85.0
print(f"Average: {result}")  # Step 7: Output: Average: 85.0
```

**Trace Table:**

| Step | score1 | score2 | score3 | total | average | result | Output |
|------|--------|--------|--------|-------|---------|--------|--------|
| 1    | 85     | -      | -      | -     | -       | -      | -      |
| 2    | 85     | 92     | -      | -     | -       | -      | -      |
| 3    | 85     | 92     | 78     | -     | -       | -      | -      |
| 4    | 85     | 92     | 78     | 255   | -       | -      | -      |
| 5    | 85     | 92     | 78     | 255   | 85.0    | -      | -      |
| 6    | -      | -      | -      | -     | -       | 85.0   | -      |
| 7    | -      | -      | -      | -     | -       | 85.0   | Average: 85.0 |

### Example 2: Tracing with Global and Local Variables

```python
# Global variables
tax_rate = 0.20
currency = "GBP"

def calculate_total(price):
    # Local variables
    tax_amount = price * tax_rate  # Uses global tax_rate
    total = price + tax_amount
    return total

# Main program
item_price = 100.0
final_price = calculate_total(item_price)
print(f"Total: {currency} {final_price}")
```

**Trace Table:**

| Step | tax_rate (G) | currency (G) | price (L) | tax_amount (L) | total (L) | item_price | final_price | Output |
|------|--------------|--------------|-----------|----------------|-----------|------------|-------------|--------|
| 1    | 0.20         | "GBP"        | -         | -              | -         | -          | -           | -      |
| 2    | 0.20         | "GBP"        | -         | -              | -         | 100.0      | -           | -      |
| 3    | 0.20         | "GBP"        | 100.0     | -              | -         | 100.0      | -           | -      |
| 4    | 0.20         | "GBP"        | 100.0     | 20.0           | -         | 100.0      | -           | -      |
| 5    | 0.20         | "GBP"        | 100.0     | 20.0           | 120.0     | 100.0      | -           | -      |
| 6    | 0.20         | "GBP"        | -         | -              | -         | 100.0      | 120.0       | -      |
| 7    | 0.20         | "GBP"        | -         | -              | -         | 100.0      | 120.0       | Total: GBP 120.0 |

Note: (G) = Global variable, (L) = Local variable

## Practical Example: Student Grade Calculator

Here's a complete program demonstrating all concepts together:

```python
# Global variables for grade boundaries
GRADE_A = 80
GRADE_B = 70
GRADE_C = 60
GRADE_D = 50

def calculate_grade(score):
    """
    Determines letter grade based on numerical score.
    Uses global grade boundaries and local variables.
    """
    # Local variable for the grade
    grade = ""
    
    # Determine grade using global boundaries
    if score >= GRADE_A:
        grade = "A"
    elif score >= GRADE_B:
        grade = "B"
    elif score >= GRADE_C:
        grade = "C"
    elif score >= GRADE_D:
        grade = "D"
    else:
        grade = "F"
    
    return grade

def process_student_scores(student_name, scores):
    """
    Processes a list of scores for a student.
    Demonstrates iteration with for loops.
    """
    # Local variables
    total = 0
    count = 0
    
    # Use for loop to iterate through scores
    for score in scores:
        total = total + score
        count = count + 1
    
    # Calculate average
    average = total / count
    
    # Get letter grade
    letter_grade = calculate_grade(average)
    
    # Display results
    print(f"\nStudent: {student_name}")
    print(f"Scores: {scores}")
    print(f"Average: {average:.1f}")
    print(f"Grade: {letter_grade}")
    
    return average, letter_grade

# Main program
student1_scores = [85, 92, 78, 88, 90]
student2_scores = [65, 72, 68, 70, 75]

# Process each student
avg1, grade1 = process_student_scores("Alice Johnson", student1_scores)
avg2, grade2 = process_student_scores("Bob Smith", student2_scores)

# Compare students
print("\n--- Summary ---")
if avg1 > avg2:
    print("Alice has the higher average")
else:
    print("Bob has the higher average")
```

**Output:**
```
Student: Alice Johnson
Scores: [85, 92, 78, 88, 90]
Average: 86.6
Grade: A

Student: Bob Smith
Scores: [65, 72, 68, 70, 75]
Average: 70.0
Grade: B

--- Summary ---
Alice has the higher average
```

## Common Pitfalls and Tips

### Type Mismatches

```python
# WRONG: Mixing incompatible types
age = "17"  # String, not integer
next_year = age + 1  # Error! Can't add string and integer

# CORRECT: Convert types appropriately
age = "17"
next_year = int(age) + 1  # Convert string to integer first
print(next_year)  # Output: 18
```

When type mismatches occur, Python raises exceptions. Learn more about handling these errors in [[B2.1.3_Exceptions|exception handling]].

### Variable Naming Conventions

```python
# GOOD: Descriptive, lowercase with underscores
student_count = 30
average_score = 85.5
is_valid = True

# AVOID: Unclear or inconsistent names
x = 30  # What does x represent?
StudentCount = 30  # Inconsistent with Python conventions
avg_scr = 85.5  # Unclear abbreviation
```

### Unintended Global Variable Modification

```python
# Global variable
balance = 1000

def withdraw(amount):
    # WRONG: Forgot 'global' keyword
    balance = balance - amount  # Error! Local variable referenced before assignment
    return balance

# CORRECT: Use global keyword
def withdraw_correct(amount):
    global balance
    balance = balance - amount
    return balance
```

When you encounter errors like these, use [[B2.1.4_Debugging|debugging techniques]] such as print statements and trace tables to identify the problem.

## Summary

- **Variables** store data in named containers with specific data types
- **Data types** include: integer, float, string, Boolean, and character (string of length 1)
- **Local variables** exist only within their function and are destroyed when the function ends
- **Global variables** exist throughout the program and can be accessed anywhere
- **Tracing** helps you understand how variable values change during program execution
- Always use **for loops** for iteration in this course
- Choose meaningful variable names and be mindful of scope to write clear, maintainable code

Understanding variables and their scope is fundamental to all programming. These concepts connect directly to how the [[Central_Processing_Unit|CPU]] manages data in registers and how the [[Operating_System|operating system]] allocates memory for your programs.
