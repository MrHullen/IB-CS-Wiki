> B2.1.4 Construct and use common debugging techniques. Techniques may include: trace tables, breakpoint debugging, print statements, step-by-step execution.

## Introduction to Debugging

Debugging is the process of identifying and fixing errors (bugs) in your code. Even experienced programmers make mistakes, so learning effective debugging techniques is essential for becoming a proficient developer. Think of debugging as detective work—you're gathering clues to solve the mystery of why your program isn't behaving as expected.

When working with [[B2.1.1_Variables|variables]], [[B2.1.2_Substrings|string manipulation]], or [[B2.1.3_Exceptions|exception handling]], bugs can creep in at any stage. The techniques covered here will help you systematically track down and resolve these issues.

## Common Types of Bugs

Before diving into debugging techniques, it's helpful to understand the main categories of errors:

- **Syntax Errors**: Mistakes in the code structure that prevent the program from running (e.g., missing colons, incorrect indentation)
- **Runtime Errors**: Errors that occur whilst the program is executing (e.g., division by zero, accessing an invalid index)
- **Logic Errors**: The program runs without crashing, but produces incorrect results due to flawed logic

## Debugging Technique 1: Print Statements

The simplest and most widely used debugging technique is inserting print statements to display variable values and program flow.

### How It Works

By strategically placing `print()` statements throughout your code, you can observe:
- The values of [[B2.1.1_Variables|variables]] at different points in execution
- Whether certain code blocks are being executed
- The order in which operations occur

### Example: Debugging a List Sum Function

```python
def calculate_average(numbers):
    """Calculate the average of a list of numbers."""
    total = 0
    
    # Print statement to check input
    print(f"Input list: {numbers}")
    
    # Using a for loop to sum the numbers
    for num in numbers:
        total += num
        # Print statement to track running total
        print(f"Added {num}, running total: {total}")
    
    average = total / len(numbers)
    
    # Print statement to check final result
    print(f"Final average: {average}")
    
    return average

# Test the function
scores = [85, 92, 78, 90]
result = calculate_average(scores)
print(f"Result: {result}")
```

**Output:**
```
Input list: [85, 92, 78, 90]
Added 85, running total: 85
Added 92, running total: 177
Added 78, running total: 255
Added 90, running total: 345
Final average: 86.25
Result: 86.25
```

### Advantages and Limitations

**Advantages:**
- Quick and easy to implement
- Works in any programming environment
- Useful for understanding program flow

**Limitations:**
- Can clutter your code with temporary statements
- Must be manually removed after debugging
- Not ideal for complex programs with many variables

## Debugging Technique 2: Trace Tables

A trace table is a systematic way to track the values of [[B2.1.1_Variables|variables]] as your program executes, line by line. This technique is particularly useful for understanding loops and conditional logic.

### How to Create a Trace Table

1. List all relevant variables as column headers
2. Add a column for the line number or operation
3. Record the value of each variable after each significant operation
4. Track the program flow through iterations

### Example: Tracing a Search Algorithm

```python
def find_maximum(numbers):
    """Find the maximum value in a list using a for loop."""
    max_value = numbers[0]  # Assume first element is maximum
    
    for i in numbers:
        if i > max_value:
            max_value = i
    
    return max_value

# Test data
data = [23, 67, 12, 89, 45]
result = find_maximum(data)
```

**Trace Table:**

| Line | Operation | i | max_value | Condition (i > max_value) |
|------|-----------|---|-----------|---------------------------|
| 2 | Initialise | - | 23 | - |
| 4 | First iteration | 23 | 23 | False (23 > 23) |
| 4 | Second iteration | 67 | 23 | True (67 > 23) |
| 5 | Update max_value | 67 | 67 | - |
| 4 | Third iteration | 12 | 67 | False (12 > 67) |
| 4 | Fourth iteration | 89 | 67 | True (89 > 67) |
| 5 | Update max_value | 89 | 89 | - |
| 4 | Fifth iteration | 45 | 89 | False (45 > 89) |
| 7 | Return | - | 89 | - |

### Advantages and Limitations

**Advantages:**
- Provides a clear, visual representation of program execution
- Excellent for understanding loop behaviour
- Helps identify logic errors
- Useful for exam preparation and algorithm analysis

**Limitations:**
- Time-consuming for large datasets
- Impractical for programs with many variables
- Manual process prone to human error

## Debugging Technique 3: Breakpoint Debugging

Breakpoint debugging allows you to pause program execution at specific lines and inspect the state of your program. This is done using an Integrated Development Environment (IDE) such as PyCharm, VS Code, or IDLE.

### How It Works

1. Set a **breakpoint** on a line where you want execution to pause
2. Run the program in debug mode
3. When execution reaches the breakpoint, the program pauses
4. Inspect variable values, the call stack, and program state
5. Step through the code line by line or continue to the next breakpoint

### Example: Using Breakpoints

```python
def process_grades(student_grades):
    """Process student grades and calculate statistics."""
    passing_grades = []
    failing_grades = []
    
    # Set a breakpoint here to inspect the input
    for grade in student_grades:
        # Set a breakpoint here to check each iteration
        if grade >= 50:
            passing_grades.append(grade)
        else:
            failing_grades.append(grade)
    
    # Set a breakpoint here to verify final lists
    pass_rate = len(passing_grades) / len(student_grades) * 100
    
    return {
        'passing': passing_grades,
        'failing': failing_grades,
        'pass_rate': pass_rate
    }

# Test the function
grades = [45, 78, 62, 38, 91, 55]
results = process_grades(grades)
print(results)
```

### Common Breakpoint Operations

- **Step Over**: Execute the current line and move to the next line
- **Step Into**: If the current line contains a function call, enter that function
- **Step Out**: Complete execution of the current function and return to the caller
- **Continue**: Resume normal execution until the next breakpoint

### Advantages and Limitations

**Advantages:**
- Allows real-time inspection of program state
- Can examine complex data structures easily
- No need to modify code with print statements
- Can conditionally break based on variable values

**Limitations:**
- Requires an IDE with debugging capabilities
- Learning curve for using debugger features
- Can be slower than other methods for simple bugs

## Debugging Technique 4: Step-by-Step Execution

Step-by-step execution (also called stepping) is closely related to breakpoint debugging. It involves executing your program one line at a time, observing how each statement affects the program state.

### When to Use Step-by-Step Execution

- When you need to understand complex logic flow
- When debugging nested loops or conditional statements
- When tracking how [[B2.1.1_Variables|variables]] change through multiple operations
- When investigating [[B2.1.3_Exceptions|exception handling]] behaviour

### Example: Debugging String Manipulation

```python
def extract_initials(full_name):
    """Extract initials from a full name."""
    # Split the name into parts
    name_parts = full_name.split()
    
    initials = ""
    
    # Use a for loop to process each part
    for part in name_parts:
        # Extract first character and make it uppercase
        initial = part[0].upper()
        initials += initial
    
    return initials

# Test with step-by-step execution
name = "john ronald reuel tolkien"
result = extract_initials(name)
print(f"Initials: {result}")
```

By stepping through this code, you can observe:
1. How `split()` creates the list `['john', 'ronald', 'reuel', 'tolkien']`
2. Each iteration of the for loop
3. How [[B2.1.2_Substrings|substring extraction]] with `[0]` works
4. The gradual building of the `initials` string

## Combining Debugging Techniques

Professional developers often combine multiple debugging techniques:

```python
def validate_and_process_data(data_list):
    """Validate and process a list of numerical data."""
    processed = []
    errors = []
    
    # Print statement to confirm input
    print(f"Processing {len(data_list)} items")
    
    # Use a for loop with enumeration for better debugging
    for index, value in enumerate(data_list):
        try:
            # Set a breakpoint here to inspect each value
            num = float(value)
            
            # Trace table would track: index, value, num, processed
            if num >= 0:
                processed.append(num * 2)
                print(f"Index {index}: Processed {value} -> {num * 2}")
            else:
                errors.append(f"Negative value at index {index}: {num}")
                print(f"Index {index}: Rejected negative value {num}")
                
        except ValueError:
            # Exception handling with debugging output
            errors.append(f"Invalid value at index {index}: {value}")
            print(f"Index {index}: Could not convert '{value}' to number")
    
    # Print final summary
    print(f"Successfully processed: {len(processed)} items")
    print(f"Errors encountered: {len(errors)} items")
    
    return processed, errors

# Test data with intentional issues
test_data = ["42", "3.14", "invalid", "-5", "100"]
valid, invalid = validate_and_process_data(test_data)
```

## Best Practices for Debugging

1. **Reproduce the Bug**: Ensure you can consistently trigger the error before attempting to fix it
2. **Isolate the Problem**: Narrow down which section of code is causing the issue
3. **Form a Hypothesis**: Based on the symptoms, predict what might be wrong
4. **Test Your Hypothesis**: Use debugging techniques to verify or refute your theory
5. **Fix and Verify**: Make the correction and thoroughly test to ensure the bug is resolved
6. **Document**: Comment your code to explain complex logic and prevent future bugs

## Common Debugging Scenarios

### Scenario 1: Off-by-One Errors in Loops

```python
def print_first_n_items(items, n):
    """Print the first n items from a list."""
    # Bug: This will cause an IndexError if n equals len(items)
    for i in range(n):
        print(f"Item {i}: {items[i]}")

# Debug using trace table or print statements
numbers = [10, 20, 30, 40, 50]
print_first_n_items(numbers, 5)  # Will crash!

# Fixed version
def print_first_n_items_fixed(items, n):
    """Print the first n items from a list (corrected)."""
    # Ensure we don't exceed list bounds
    for i in range(min(n, len(items))):
        print(f"Item {i}: {items[i]}")
```

### Scenario 2: Logic Errors in Conditionals

```python
def categorise_temperature(temp):
    """Categorise temperature (buggy version)."""
    # Bug: Logic error in conditions
    if temp > 30:
        return "Hot"
    if temp > 20:
        return "Warm"
    if temp > 10:
        return "Cool"
    else:
        return "Cold"

# Use print statements to debug
print(categorise_temperature(25))  # Should be "Warm" but returns "Hot"

# Fixed version with elif
def categorise_temperature_fixed(temp):
    """Categorise temperature (corrected)."""
    if temp > 30:
        return "Hot"
    elif temp > 20:
        return "Warm"
    elif temp > 10:
        return "Cool"
    else:
        return "Cold"
```

## Summary

Debugging is an essential skill that improves with practice. The four main techniques covered are:

1. **Print Statements**: Quick and simple, ideal for tracking variable values
2. **Trace Tables**: Systematic approach for understanding program flow
3. **Breakpoint Debugging**: Powerful IDE-based technique for complex issues
4. **Step-by-Step Execution**: Detailed examination of code execution

By mastering these techniques and combining them effectively, you'll be able to identify and resolve bugs efficiently, leading to more robust and reliable programs.
