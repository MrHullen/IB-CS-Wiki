## Introduction to ML Hardware

Machine learning models require significant computational power, especially during training. The hardware you need depends on several factors:

- **Model complexity**: Deeper neural networks require more processing power
- **Dataset size**: Larger datasets take longer to process
- **Training vs inference**: Training is more demanding than making predictions
- **Real-time requirements**: Some applications need instant responses
- **Budget constraints**: Advanced hardware can be expensive

Understanding hardware requirements helps you choose the right tools for your [[A4.1.1_Types|machine learning]] projects.

## Standard Laptop Configurations

For learning machine learning fundamentals and working with small datasets, a standard laptop is often sufficient.

### Typical Specifications

**Minimum requirements:**
- **CPU**: Modern multi-core processor (Intel i5/i7 or AMD Ryzen 5/7)
- **RAM**: 8GB (16GB recommended)
- **Storage**: 256GB SSD
- **GPU**: Integrated graphics (Intel/AMD)

**Recommended for better performance:**
- **CPU**: Intel i7/i9 or AMD Ryzen 7/9
- **RAM**: 16-32GB
- **Storage**: 512GB+ SSD
- **GPU**: Dedicated GPU with 4-8GB VRAM (NVIDIA GTX/RTX series)

### Suitable Use Cases

Standard laptops work well for:
- Learning ML concepts and algorithms
- Prototyping small models
- Working with datasets under 10,000 samples
- Running pre-trained models for inference
- Basic image classification tasks
- Simple natural language processing

## Graphics Processing Units (GPUs)

GPUs are specialised processors originally designed for rendering graphics, but they excel at the parallel computations required for [[A4.1.1_Types|deep learning]].

### Why GPUs for Machine Learning?

**Parallel processing advantage:**
- [[Central_Processing_Unit|CPUs]] have 4-16 cores optimised for sequential tasks
- GPUs have thousands of smaller cores optimised for parallel operations
- Neural network training involves many identical calculations on different data
- GPUs can process multiple data points simultaneously

**Performance comparison:**
- Training a model on GPU: Hours
- Same model on CPU: Days or weeks
- Speed-up factor: 10-100x faster

### Popular GPU Options

**Consumer GPUs (NVIDIA GeForce):**
- RTX 3060: 12GB VRAM, good for learning
- RTX 3080: 10-12GB VRAM, excellent for serious projects
- RTX 4090: 24GB VRAM, professional-grade performance

**Professional GPUs (NVIDIA):**
- A100: 40-80GB VRAM, data centre grade
- H100: Latest generation, extreme performance
- V100: Previous generation, still widely used

### GPU Memory Considerations

```python
# Example: Calculating GPU memory requirements
def estimate_gpu_memory(model_params, batch_size, precision="float32"):
    """
    Estimate GPU memory needed for training
    
    Args:
        model_params: Number of model parameters (millions)
        batch_size: Number of samples processed together
        precision: Data type precision
    """
    # Bytes per parameter based on precision
    bytes_per_param = {
        "float32": 4,  # 32 bits = 4 bytes
        "float16": 2,  # 16 bits = 2 bytes (mixed precision)
        "int8": 1      # 8 bits = 1 byte (quantised)
    }
    
    # Memory components (simplified)
    model_memory = model_params * 1_000_000 * bytes_per_param[precision]
    
    # Gradients (same size as model for backpropagation)
    gradient_memory = model_memory
    
    # Optimiser states (e.g., Adam stores momentum)
    optimiser_memory = model_memory * 2
    
    # Activations (depends on batch size)
    activation_memory = batch_size * model_params * 1000 * bytes_per_param[precision]
    
    # Total memory in GB
    total_bytes = model_memory + gradient_memory + optimiser_memory + activation_memory
    total_gb = total_bytes / (1024 ** 3)
    
    return {
        "model_gb": model_memory / (1024 ** 3),
        "gradients_gb": gradient_memory / (1024 ** 3),
        "optimiser_gb": optimiser_memory / (1024 ** 3),
        "activations_gb": activation_memory / (1024 ** 3),
        "total_gb": total_gb
    }

# Example calculations
print("GPU Memory Requirements:\n")

models = [
    {"name": "Small CNN", "params": 1, "batch": 32},
    {"name": "ResNet-50", "params": 25, "batch": 32},
    {"name": "GPT-2", "params": 117, "batch": 8},
    {"name": "Large Transformer", "params": 1000, "batch": 4}
]

for model in models:
    memory = estimate_gpu_memory(model["params"], model["batch"])
    print(f"{model['name']} ({model['params']}M parameters, batch size {model['batch']}):")
    print(f"  Model weights: {memory['model_gb']:.2f} GB")
    print(f"  Gradients: {memory['gradients_gb']:.2f} GB")
    print(f"  Optimiser: {memory['optimiser_gb']:.2f} GB")
    print(f"  Activations: {memory['activations_gb']:.2f} GB")
    print(f"  Total required: {memory['total_gb']:.2f} GB")
    
    # Recommend GPU
    if memory['total_gb'] < 8:
        print(f"  ✓ Fits on: RTX 3060 (12GB)")
    elif memory['total_gb'] < 12:
        print(f"  ✓ Fits on: RTX 3080 (12GB)")
    elif memory['total_gb'] < 24:
        print(f"  ✓ Fits on: RTX 4090 (24GB)")
    else:
        print(f"  ⚠ Requires: Professional GPU (A100 40-80GB)")
    print()
```

## Tensor Processing Units (TPUs)

TPUs are specialised processors designed by Google specifically for machine learning workloads, particularly neural network training and inference.

### TPU Architecture

**Key features:**
- Custom-built for tensor operations (matrix multiplications)
- Optimised for TensorFlow and JAX frameworks
- Extremely high throughput for specific operations
- Lower precision arithmetic (bfloat16) for efficiency

**Performance characteristics:**
- Excellent for large-scale training
- Optimised for batch processing
- May not be ideal for all model architectures
- Best performance with models designed for TPU architecture

### TPU vs GPU

| Feature | GPU | TPU |
|---------|-----|-----|
| **Flexibility** | Works with any framework | Optimised for TensorFlow/JAX |
| **Precision** | float32, float16, int8 | bfloat16 primarily |
| **Use cases** | General ML, gaming, graphics | Large-scale neural networks |
| **Availability** | Widely available | Google Cloud Platform |
| **Cost** | Purchase or rent | Cloud rental only |

### When to Use TPUs

## Application-Specific Integrated Circuits (ASICs)

ASICs are custom-designed chips built for specific tasks. In machine learning, they're optimised for particular model architectures or operations.

### Characteristics

**Advantages:**
- Maximum efficiency for specific tasks
- Lower power consumption than general-purpose hardware
- Highest performance for targeted operations
- Cost-effective at scale

**Disadvantages:**
- Cannot be reprogrammed for different tasks
- High development costs
- Long development time
- Inflexible to algorithm changes

### Real-World Examples

**Google TPU**: ASIC designed for neural network operations
**Bitcoin miners**: ASICs for cryptocurrency mining (similar concept)
**Inference accelerators**: Chips optimised for running trained models

### Use Cases

```python
# Example: When ASICs make sense
def evaluate_asic_viability(deployment_scale, model_stability, budget):
    """
    Determine if developing an ASIC is worthwhile
    
    Args:
        deployment_scale: Number of devices needed
        model_stability: How often the model changes
        budget: Development budget available
    """
    # ASIC development costs (millions)
    development_cost = 5_000_000
    
    # Cost per unit
    asic_unit_cost = 50
    gpu_unit_cost = 500
    
    # Calculate total costs
    asic_total = development_cost + (deployment_scale * asic_unit_cost)
    gpu_total = deployment_scale * gpu_unit_cost
    
    # Power consumption (watts)
    asic_power = 10
    gpu_power = 250
    
    # Annual power cost per device (assuming £0.15/kWh, 24/7 operation)
    hours_per_year = 24 * 365
    power_cost_per_kwh = 0.15
    
    asic_annual_power = (asic_power / 1000) * hours_per_year * power_cost_per_kwh
    gpu_annual_power = (gpu_power / 1000) * hours_per_year * power_cost_per_kwh
    
    # 5-year total cost of ownership
    years = 5
    asic_tco = asic_total + (deployment_scale * asic_annual_power * years)
    gpu_tco = gpu_total + (deployment_scale * gpu_annual_power * years)
    
    return {
        "asic_initial": asic_total,
        "gpu_initial": gpu_total,
        "asic_tco": asic_tco,
        "gpu_tco": gpu_tco,
        "savings": gpu_tco - asic_tco,
        "breakeven_units": development_cost / (gpu_unit_cost - asic_unit_cost),
        "recommended": "ASIC" if asic_tco < gpu_tco and model_stability == "stable" else "GPU"
    }

# Analyse different deployment scenarios
print("ASIC vs GPU Economic Analysis:\n")

scenarios = [
    {"name": "Small startup", "scale": 100, "stability": "changing"},
    {"name": "Medium company", "scale": 10_000, "stability": "stable"},
    {"name": "Large deployment", "scale": 100_000, "stability": "stable"}
]

for scenario in scenarios:
    analysis = evaluate_asic_viability(
        scenario["scale"],
        scenario["stability"],
        budget=10_000_000
    )
    
    print(f"{scenario['name']} ({scenario['scale']:,} devices):")
    print(f"  Model stability: {scenario['stability']}")
    print(f"  ASIC 5-year cost: £{analysis['asic_tco']:,.0f}")
    print(f"  GPU 5-year cost: £{analysis['gpu_tco']:,.0f}")
    print(f"  Potential savings: £{analysis['savings']:,.0f}")
    print(f"  Break-even point: {analysis['breakeven_units']:,.0f} devices")
    print(f"  → Recommendation: {analysis['recommended']}")
    print()
```

## Field-Programmable Gate Arrays (FPGAs)

FPGAs are reconfigurable chips that can be programmed to perform specific operations. They offer a middle ground between flexible GPUs and specialised ASICs.

### Key Characteristics

**Flexibility:**
- Can be reprogrammed for different tasks
- Adapt to algorithm changes
- Update functionality after deployment

**Performance:**
- Faster than GPUs for specific operations
- Lower latency than GPUs
- More efficient than general-purpose processors

**Power efficiency:**
- Lower power consumption than GPUs
- Higher efficiency than CPUs
- Suitable for edge deployment

### Advantages and Trade-offs

| Aspect | FPGA | GPU | ASIC |
|--------|------|-----|------|
| **Flexibility** | High (reprogrammable) | High | None |
| **Performance** | Medium-High | High | Highest |
| **Power efficiency** | High | Medium | Highest |
| **Development time** | Weeks-Months | Days | Years |
| **Cost per unit** | Medium | Medium | Low (at scale) |
| **Latency** | Very low | Low | Very low |

### Use Cases

## Edge Devices

Edge devices are compact, power-efficient hardware deployed close to where data is generated, enabling local processing without cloud connectivity.

### Types of Edge Devices

**Mobile devices:**
- Smartphones and tablets
- Neural processing units (NPUs) in mobile chips
- Apple Neural Engine, Qualcomm AI Engine

**Embedded systems:**
- Raspberry Pi with AI accelerators
- NVIDIA Jetson series
- Google Coral Edge TPU

**IoT devices:**
- Smart cameras
- Industrial sensors
- Wearable devices

### Why Edge Computing?

**Advantages:**
- **Low latency**: Process data locally without network delays
- **Privacy**: Sensitive data stays on device
- **Reliability**: Works without internet connection
- **Bandwidth**: Reduces data transmission costs
- **Real-time**: Immediate responses for critical applications

**Challenges:**
- Limited computational power
- Constrained memory
- Power consumption constraints
- Model size restrictions

### Edge Deployment Strategies

```python
# Example: Optimising models for edge deployment
def optimise_for_edge(model_size_mb, target_device):
    """
    Demonstrate model optimisation techniques for edge devices
    
    Args:
        model_size_mb: Original model size in megabytes
        target_device: Type of edge device
    """
    device_constraints = {
        "smartphone": {"max_size_mb": 50, "max_latency_ms": 100, "power": "battery"},
        "raspberry_pi": {"max_size_mb": 100, "max_latency_ms": 500, "power": "limited"},
        "jetson_nano": {"max_size_mb": 200, "max_latency_ms": 50, "power": "moderate"},
        "coral_tpu": {"max_size_mb": 30, "max_latency_ms": 20, "power": "very_limited"}
    }
    
    constraints = device_constraints[target_device]
    optimisations = []
    size_reduction = 0
    
    print(f"Optimising {model_size_mb}MB model for {target_device}")
    print(f"Target constraints: {constraints['max_size_mb']}MB, {constraints['max_latency_ms']}ms latency\n")
    
    current_size = model_size_mb
    
    # Quantisation: Convert float32 to int8
    if current_size > constraints["max_size_mb"]:
        quantisation_reduction = current_size * 0.75  # 4x smaller (32-bit to 8-bit)
        current_size -= quantisation_reduction
        size_reduction += quantisation_reduction
        optimisations.append({
            "technique": "Quantisation (float32 → int8)",
            "reduction_mb": quantisation_reduction,
            "new_size_mb": current_size,
            "accuracy_impact": "~1-2% loss"
        })
    
    # Pruning: Remove unnecessary connections
    if current_size > constraints["max_size_mb"]:
        pruning_reduction = current_size * 0.3  # Remove 30% of weights
        current_size -= pruning_reduction
        size_reduction += pruning_reduction
        optimisations.append({
            "technique": "Weight pruning (30%)",
            "reduction_mb": pruning_reduction,
            "new_size_mb": current_size,
            "accuracy_impact": "~0.5-1% loss"
        })
    
    # Knowledge distillation: Train smaller model
    if current_size > constraints["max_size_mb"]:
        distillation_reduction = current_size * 0.5  # 50% smaller student model
        current_size -= distillation_reduction
        size_reduction += distillation_reduction
        optimisations.append({
            "technique": "Knowledge distillation",
            "reduction_mb": distillation_reduction,
            "new_size_mb": current_size,
            "accuracy_impact": "~2-3% loss"
        })
    
    # Display results
    print("Optimisation steps:")
    for i, opt in enumerate(optimisations, 1):
        print(f"\n{i}. {opt['technique']}")
        print(f"   Size reduction: {opt['reduction_mb']:.1f}MB")
        print(f"   New size: {opt['new_size_mb']:.1f}MB")
        print(f"   Accuracy impact: {opt['accuracy_impact']}")
    
    print(f"\nFinal results:")
    print(f"  Original size: {model_size_mb}MB")
    print(f"  Optimised size: {current_size:.1f}MB")
    print(f"  Total reduction: {size_reduction:.1f}MB ({(size_reduction/model_size_mb)*100:.1f}%)")
    print(f"  Fits on {target_device}: {'✓ Yes' if current_size <= constraints['max_size_mb'] else '✗ No'}")
    
    return current_size
```

```python
# Example: Cost comparison cloud vs on-premise
def compare_deployment_costs(training_hours_per_month, inference_requests_per_month):
    """
    Compare costs of cloud vs on-premise deployment
    
    Args:
        training_hours_per_month: Hours of GPU training needed monthly
        inference_requests_per_month: Number of inference requests monthly
    """
    # On-premise costs
    gpu_purchase_cost = 2000  # High-end GPU
    server_cost = 3000
    maintenance_annual = 1000
    power_annual = 500  # Electricity
    
    on_premise_initial = gpu_purchase_cost + server_cost
    on_premise_annual = maintenance_annual + power_annual
    
    # Cloud costs (approximate)
    gpu_instance_hourly = 3.00  # AWS p3.2xlarge
    inference_per_1000 = 0.10  # Serverless inference
    
    cloud_training_monthly = training_hours_per_month * gpu_instance_hourly
    cloud_inference_monthly = (inference_requests_per_month / 1000) * inference_per_1000
    cloud_monthly = cloud_training_monthly + cloud_inference_monthly
    cloud_annual = cloud_monthly * 12
    
    # Calculate break-even
    years_to_breakeven = on_premise_initial / (cloud_annual - on_premise_annual)
    
    print("Cost Comparison: Cloud vs On-Premise\n")
    print(f"Usage: {training_hours_per_month}hrs training/month, {inference_requests_per_month:,} inferences/month\n")
    
    print("On-Premise:")
    print(f"  Initial investment: £{on_premise_initial:,.0f}")
    print(f"  Annual operating cost: £{on_premise_annual:,.0f}")
    print(f"  3-year total: £{on_premise_initial + (on_premise_annual * 3):,.0f}")
    
    print("\nCloud:")

    print(f"  Annual cost: £{cloud_annual:,.2f}")

    return {

        "cloud_3yr": cloud_annual * 3,
        "breakeven_years": years_to_breakeven
    }
```

## High-Performance Computing (HPC) Centres

HPC centres are specialised facilities with massive computational resources, designed for the most demanding scientific and research workloads.

### Characteristics

**Scale:**
- Thousands of GPUs/TPUs

```python
# Example: Simulating distributed training across HPC cluster
def simulate_distributed_training(model_params_billions, num_gpus, dataset_size_tb):
    """
    Simulate training time on HPC cluster with distributed training
    
    Args:
        model_params_billions: Model size in billions of parameters
        num_gpus: Number of GPUs in cluster
        dataset_size_tb: Dataset size in terabytes
    """
    # Base training time on single GPU (days)
    single_gpu_days = model_params_billions * dataset_size_tb * 10
    
    # Scaling efficiency (not perfectly linear due to communication overhead)
    def scaling_efficiency(n_gpus):
        """Calculate efficiency loss from inter-GPU communication"""
        if n_gpus <= 8:
            return 0.95  # 95% efficient
        elif n_gpus <= 64:
            return 0.85  # 85% efficient
        elif n_gpus <= 512:
            return 0.70  # 70% efficient
        else:
            return 0.60  # 60% efficient
    
    efficiency = scaling_efficiency(num_gpus)
    distributed_days = single_gpu_days / (num_gpus * efficiency)
    
    # Calculate costs
    gpu_hour_cost = 3.00
    total_gpu_hours = distributed_days * 24 * num_gpus
    total_cost = total_gpu_hours * gpu_hour_cost
    
    print(f"Distributed Training Simulation:")
    print(f"  Model size: {model_params_billions}B parameters")
    print(f"  Dataset: {dataset_size_tb}TB")
    print(f"  Cluster: {num_gpus} GPUs")
    print(f"\nResults:")
    print(f"  Single GPU time: {single_gpu_days:.1f} days")
    print(f"  Distributed time: {distributed_days:.1f} days ({distributed_days * 24:.1f} hours)")
    print(f"  Scaling efficiency: {efficiency * 100:.0f}%")
    print(f"  Speed-up: {single_gpu_days / distributed_days:.1f}x")
    print(f"  Total GPU-hours: {total_gpu_hours:,.0f}")
    print(f"  Estimated cost: £{total_cost:,.0f}")
    
    return distributed_days
```

### Memory Hierarchy

Understanding memory types helps optimise performance:

| Memory Type | Size | Speed | Use |
|-------------|------|-------|-----|
| **Registers** | Bytes | Fastest | Immediate operations |
| **L1 Cache** | KB | Very fast | Active data |
| **L2/L3 Cache** | MB | Fast | Recent data |
| **RAM** | GB | Moderate | Working dataset |
| **GPU VRAM** | GB | Fast (parallel) | Model + batch |
| **SSD** | TB | Slower | Full dataset |
| **HDD** | TB | Slowest | Archives |
