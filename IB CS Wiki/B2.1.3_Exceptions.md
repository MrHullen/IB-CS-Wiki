## What is an Exception?

An **exception** is an error that occurs during program execution, disrupting the normal flow of instructions. When Python encounters an error it cannot handle, it "raises" (or "throws") an exception, which typically causes the program to crash unless we handle it properly.

Think of exceptions like unexpected obstacles on a journey—if you don't plan for them, they'll stop you in your tracks. But with proper exception handling, your program can gracefully recover or provide helpful error messages instead of crashing.

Common scenarios that cause exceptions:
- Dividing by zero
- Accessing a file that doesn't exist
- Converting invalid [[B2.1.1_Variables|string data]] to a number
- Accessing an index that's out of range in a list
- Network connection failures

## Why Exception Handling Matters

Without exception handling, a single error can crash your entire program. Consider this example:

```python
# Program without exception handling - CRASHES!
age_input = input("Enter your age: ")
age = int(age_input)  # What if user enters "twenty"?
print(f"You are {age} years old")
print("Program continues...")  # This line never runs if there's an error
```

If the user enters `"twenty"` instead of `20`, Python raises a `ValueError` and the program terminates immediately. The message "Program continues..." never appears.

## The try/except Structure

Python uses `try` and `except` blocks to handle exceptions gracefully:

```python
try:
    # Code that might cause an exception
    risky_operation()
except ExceptionType:
    # Code to handle the exception
    handle_error()
```

### Basic Example

```python
# Program with exception handling - ROBUST!
try:
    age_input = input("Enter your age: ")
    age = int(age_input)
    print(f"You are {age} years old")
except ValueError:
    print("Error: Please enter a valid number for age")

print("Program continues...")  # This ALWAYS runs
```

Now if the user enters invalid input, the program displays a helpful error message and continues running.

## Common Exception Types

Python has many built-in exception types. Here are the most common ones you'll encounter:

| Exception Type | Description | Example Cause |
|----------------|-------------|---------------|
| `ValueError` | Invalid value for an operation | `int("hello")` |
| `ZeroDivisionError` | Division by zero | `10 / 0` |
| `TypeError` | Operation on incompatible types | `"5" + 5` |
| `IndexError` | Index out of range | `list[10]` when list has 5 items |
| `KeyError` | Dictionary key doesn't exist | `dict["missing_key"]` |
| `FileNotFoundError` | File doesn't exist | Opening a non-existent file |
| `AttributeError` | Attribute doesn't exist | Calling a non-existent method |

## Handling Multiple Exception Types

You can handle different exceptions in different ways:

```python
# Handling multiple specific exceptions
try:
    numbers = [10, 20, 30]
    index = int(input("Enter an index: "))
    divisor = int(input("Enter a divisor: "))
    
    result = numbers[index] / divisor
    print(f"Result: {result}")
    
except ValueError:
    print("Error: Please enter valid numbers")
except IndexError:
    print("Error: Index out of range")
except ZeroDivisionError:
    print("Error: Cannot divide by zero")
```

### Catching All Exceptions

You can use a general `except` clause to catch any exception:

```python
# Catch any exception (use sparingly!)
try:
    risky_operation()
except Exception as e:
    print(f"An error occurred: {e}")
```

**Note:** While catching all exceptions is sometimes useful, it's generally better to catch specific exceptions so you can handle each type appropriately.

## The finally Block

The `finally` block contains code that **always** executes, whether an exception occurred or not. This is crucial for cleanup operations like closing files or network connections.

```python
# Using finally for cleanup
try:
    file = open("data.txt", "r")
    content = file.read()
    print(content)
except FileNotFoundError:
    print("Error: File not found")
finally:
    # This ALWAYS runs, even if there's an error
    try:
        file.close()
        print("File closed successfully")
    except:
        print("No file to close")
```

## Practical Examples

### Example 1: Safe Division Calculator

```python
# Safe division with exception handling
def safe_divide(a, b):
    """Safely divide two numbers with error handling"""
    try:
        result = a / b
        return result
    except ZeroDivisionError:
        print("Error: Cannot divide by zero")
        return None
    except TypeError:
        print("Error: Both inputs must be numbers")
        return None

# Test the function
print(safe_divide(10, 2))    # Output: 5.0
print(safe_divide(10, 0))    # Output: Error message, then None
print(safe_divide(10, "2"))  # Output: Error message, then None
```

### Example 2: Processing User Input with Validation

```python
# Get a valid integer from the user
def get_valid_integer(prompt):
    """Keep asking until user enters a valid integer"""
    while True:
        try:
            value = int(input(prompt))
            return value  # Success! Exit the loop
        except ValueError:
            print("Invalid input. Please enter a whole number.")

# Use the function
age = get_valid_integer("Enter your age: ")
print(f"You are {age} years old")
```

**Note:** While this example uses a `while` loop for continuous validation, when processing collections of data, always use `for` loops as per our course standards.

### Example 3: Processing Multiple Files

filenames = ["data1.txt", "data2.txt", "missing.txt", "data3.txt"]

## Exception Handling Best Practices

### 1. Be Specific

Catch specific exceptions rather than using a general `except` clause:

```python
# GOOD: Specific exception handling
try:
    value = int(input("Enter a number: "))
except ValueError:
    print("That's not a valid number")

# AVOID: Too general
try:
    value = int(input("Enter a number: "))
except:
    print("Something went wrong")  # What went wrong? We don't know!
```

### 2. Don't Hide Errors Silently

Always provide feedback when an exception occurs:

```python
# BAD: Silent failure
try:
    result = risky_operation()
except:
    pass  # Error is completely hidden!

# GOOD: Informative error handling
try:
    result = risky_operation()
except Exception as e:
    print(f"Operation failed: {e}")
    result = None  # Provide a safe default value
```

### 3. Use finally for Cleanup

Always close resources like files, even if an error occurs:

```python
# GOOD: Guaranteed cleanup
file = None
try:
    file = open("data.txt", "r")
    process_data(file.read())
except FileNotFoundError:
    print("File not found")
finally:
    if file:
        file.close()

# BETTER: Use context managers (with statement)
try:
    with open("data.txt", "r") as file:
        process_data(file.read())
except FileNotFoundError:
    print("File not found")
# File automatically closed!
```

## Common Debugging Techniques

Exception handling works hand-in-hand with debugging. Here are essential debugging techniques:

### 1. Print Statements

Add strategic print statements to track program flow:

```python
def calculate_average(numbers):
    print(f"DEBUG: Received {len(numbers)} numbers")  # Debug output
    
    try:
        total = sum(numbers)
        print(f"DEBUG: Total = {total}")  # Debug output
        
        average = total / len(numbers)
        print(f"DEBUG: Average = {average}")  # Debug output
        
        return average
    except ZeroDivisionError:
        print("ERROR: Cannot calculate average of empty list")
        return 0
```

### 2. Trace Tables

Manually track [[B2.1.1_Variables|variable]] values through each step:

```python
# Example code to trace
numbers = [10, 20, 30]
total = 0

for num in numbers:
    total = total + num
    print(f"num={num}, total={total}")  # Trace output

# Trace table:
# Step | num | total
# -----|-----|------
#  1   | 10  | 10
#  2   | 20  | 30
#  3   | 30  | 60
```

### 3. Exception Messages

Use the exception object to get detailed error information:

```python
# Capture and display exception details
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(f"Exception type: {type(e).__name__}")
    print(f"Exception message: {e}")
    print("This helps us understand exactly what went wrong")
```

### 4. Incremental Testing

Test small portions of code before combining them:

```python
# Test each function independently
def read_file(filename):
    """Read and return file contents"""
    try:
        with open(filename, "r") as file:
            return file.read()
    except FileNotFoundError:
        print(f"Error: {filename} not found")
        return ""

def count_words(text):
    """Count words in text"""
    return len(text.split())

# Test read_file first
content = read_file("test.txt")
print(f"Content length: {len(content)}")

# Then test count_words
word_count = count_words(content)
print(f"Word count: {word_count}")

# Finally combine them
def process_file(filename):
    content = read_file(filename)
    return count_words(content)
```

## Real-World Scenario: Data Processing Pipeline

Here's a comprehensive example combining multiple exception handling techniques:

```python
# Process student data from multiple sources
def process_student_data(filenames):
    """Process student records from multiple files"""
    all_students = []
    errors = []
    
    # Use for loop to process each file
    for filename in filenames:
        print(f"\nProcessing {filename}...")
        
        try:
            # Attempt to open and read file
            with open(filename, "r") as file:
                lines = file.readlines()
            
            # Process each line in the file
            for line_num, line in enumerate(lines, start=1):
                try:
                    # Parse student data: "Name,Age,Grade"
                    parts = line.strip().split(",")
                    
                    if len(parts) != 3:
                        raise ValueError("Invalid format: expected 3 fields")
                    
                    name = parts[0]
                    age = int(parts[1])  # May raise ValueError
                    grade = float(parts[2])  # May raise ValueError
                    
                    # Validate data
                    if age < 0 or age > 120:
                        raise ValueError(f"Invalid age: {age}")
                    if grade < 0 or grade > 100:
                        raise ValueError(f"Invalid grade: {grade}")
                    
                    # Add valid student
                    all_students.append({
                        "name": name,
                        "age": age,
                        "grade": grade,
                        "source": filename
                    })
                
                except ValueError as e:
                    error_msg = f"{filename}, line {line_num}: {e}"
                    errors.append(error_msg)
                    print(f"  Warning: {error_msg}")
        
        except FileNotFoundError:
            error_msg = f"{filename}: File not found"
            errors.append(error_msg)
            print(f"  Error: {error_msg}")
        
        except PermissionError:
            error_msg = f"{filename}: Permission denied"
            errors.append(error_msg)
            print(f"  Error: {error_msg}")
        
        finally:
            print(f"Finished processing {filename}")
    
    # Summary
    print(f"\n{'='*50}")
    print(f"Successfully processed: {len(all_students)} students")
    print(f"Errors encountered: {len(errors)}")
    
    return all_students, errors

# Example usage
files = ["class_a.txt", "class_b.txt", "missing.txt"]
students, errors = process_student_data(files)

# Display results
print("\nValid students:")
for student in students:
    print(f"  {student['name']}: Age {student['age']}, Grade {student['grade']}")
```

## Key Takeaways

1. **Exceptions** are errors that occur during program execution
2. Use `try`/`except` blocks to handle exceptions gracefully
3. The `finally` block always executes, perfect for cleanup operations
4. Catch **specific** exceptions rather than using general exception handlers
5. Common exception types include `ValueError`, `ZeroDivisionError`, `TypeError`, `IndexError`, and `FileNotFoundError`
6. Exception handling prevents program crashes and provides better user experience
7. Combine exception handling with debugging techniques like print statements and trace tables
8. Always provide informative error messages to help users understand what went wrong
